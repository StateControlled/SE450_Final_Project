This is an explanation of how the system works and the components of the system.

Initialization
On startup, the program generates the catalogue from a list of items.
A factory method builds the items from the data retrieved from the file and adds them to an internal "database."
User operation
A user can log in to an existing account or create a new account. A unique shopping cart is associated with each account.
The user can browse a (text only) catalogue of products.
Products can be added to or removed from the user's cart.
The user can "check out" to make a purchase.
Authentication
Credit card information will be validated (simulated).
Payment will be processed (simulated).
An record of the order will be kept.
Shutdown
Any changes to the product "database" are written to a file.
Records of new orders are written to file.
New user information is written to file.
