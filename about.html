About Final Project : Shopping System

By William Berthouex, Fall Quarter 2023

Major Project Components

User Interface
A GUI allows a User to interact with the application with mouse and keyboard.
A User can log in to an existing account; create a new account; and browse products, add them to a cart, and check out.
Product Catalogue
A Catalogue Interface serves as a basis for the three Catalogues in the system.
The first is a "Catalogue" for storing product data, the second a "Database" for storing User information,
and the third an "Order Database" for storing records of all completed orders.
The Catalogues are implemented as Singleton Patterns and instantiated before the application runs.
Logging
A LogWriter class records user actions and logs errors. It is implemented as a Singleton Pattern.
The LogWriter is instantiated in a static block before the application runs.
User Authentication
A User can enter a username and password into the GUI and the application will search for a matching account in the User Database.
Adhearing to best practices, the plaintext password is never saved to disk. Only a record of the hashed password is kept.
Payment Processing
Payment processing is simulated by a PaymentProcessor class. It makes simple validation checks for some given information and returns true if the information passes the checks.
It is possible for a credit card to "fail" validation, in which case the order is cancelled.
It is purely a simulation and is not secure nor is it intended to truely represent actual credit card validation.
Order Processing
Once a User's credit card passes validation checks, a new Order is created and added to the Order Database.
The User is given an order number as confirmation.

How to run

  1. Run CompileProject.bat
  2. Program will run and show GUI
The program will not allow items to be added to the cart if a user is not logged in.
Either log in to an existing account with a username and password or create a new account.
Once an account has logged in, a message "Welcome !" will appear on the panel and items can be added to the cart.
Both cart and product catalogue are represented in the GUI as scrollable lists.
Items can be added to and removed from the cart with the corresponding buttons and the total will automatically update above the cart.
Checking out will open a new window where the user may enter credit card details - a 16-digit credit card number, an expiration date, and a security code.
If the credit card passes validation, the order will be complete!
The user will see a confirmation screen and receive an order number. The order will be added to the Order Database.
Exiting the application from the File > Exit menu or clicking the "X" in the top right will terminate the application.
Logged events will output to FinalProjectLog.log
